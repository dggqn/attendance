# 考勤系统 - 项目上下文

## 技术决策记录

### 认证方案
- 使用 Supabase Auth（不是自建 JWT）
- 用户表：`auth.users`（系统表）+ `profiles`（扩展表）
- 触发器：`handle_new_user` 自动同步
- 登录方式：邮箱+密码（用户名转邮箱格式）

### 状态管理
- Pinia：`user.ts` 管理登录状态
- 关键函数：`checkLoginStatus`, `login`, `register`, `logout`

## 项目已完成功能

### 1. 权限系统
- 三级权限：超管(super_admin)、管理员(admin)、普通用户(user)
- 部门管理：超管创建部门，新用户首次登录需选择部门
- 数据隔离：超管看全部，管理员看本部门，普通用户看自己

### 2. 考勤功能
- 快速打卡：一键上下班打卡，自动判断迟到
- 手动补录：填写详细信息提交考勤记录
- 数据导出：导出 Excel 报表

### 3. 请假/补卡
- 提交申请：补卡、病假、事假、年假等
- 审批流程：管理员/超管审批，支持审批意见
- 申请列表：查看申请状态和审批结果

### 4. 首页仪表盘
- 今日打卡状态：显示上下班打卡情况
- 考勤日历：月历视图，一眼看出哪天打卡/缺卡
- 本月统计：出勤天数、正常、迟到、缺勤统计
- 待审批提醒：管理员查看待审批申请
- 快捷入口：快速跳转到各功能页面

### 5. 个人设置
- 修改昵称和头像（支持本地图片上传）
- 修改密码

### 6. 体验优化
- 数据为空引导：空状态时提示用户下一步操作
- 批量操作：管理员可批量删除考勤记录、批量审批申请
